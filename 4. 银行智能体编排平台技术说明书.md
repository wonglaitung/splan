## 一、平台概述

### 1.1 Dify简介

Dify是一个开源的LLM应用开发平台，旨在帮助开发者快速将AI应用从原型推进到生产环境。它提供直观的无代码界面，集成复杂AI工作流、Agent能力、RAG管道、模型管理和监控等核心功能。

**核心定位**：
- **开源免费**：Apache-2.0 许可证，社区版功能完整
- **模型中立**：支持1000+模型，包括DeepSeek、Qwen、本地模型等
- **低代码构建**：可视化拖拽界面，业务人员可快速上手
- **API开放**：工作流一键发布为REST API，供传统业务系统调用

**最新版本信息**（截至2025年）：

## 三、核心功能详解

### 3.1 Workflow（工作流编排）

Workflow是Dify的核心功能，提供可视化拖拽界面构建复杂AI应用。

#### 工作流节点类型

| 类别 | 节点类型 | 功能说明 |
|-----|---------|---------|
| **基础节点** | Start | 定义输入变量（文本、文件、下拉选项等） |
| | End | 定义输出结果 |
| | Answer | 直接返回回答（用于Chatflow） |
| | LLM | 大语言模型推理节点 |
| **逻辑节点** | If/Else | 条件分支，支持IF/ELIF/ELSE多分支 |
| | Variable Assigner | 变量赋值与类型转换 |
| | Template | 文本模板拼接 |
| **数据节点** | Knowledge Retrieval | 知识库检索，返回相关文档片段 |
| | HTTP Request | 调用外部API/Webhook |
| **代码节点** | Code | 执行Python/Node.js代码 |
| **智能节点** | Question Classifier | 问题分类（LLM驱动） |
| | Agent | 自主推理节点，支持工具调用 |
| | Iteration | 循环迭代处理数组数据 |

### 3.3 API开放能力

Dify可将可视化构建的工作流一键发布为REST API，供传统业务系统调用。

#### API端点类型

| 端点 | 用途 | 特点 |
|-----|------|------|
| `POST /v1/workflows/run` | 执行工作流（无状态） | 单次执行，适合批量处理 |
| `POST /v1/chat-messages` | 对话式交互（有状态） | 保持会话上下文 |
| `POST /workflows/tasks/{task_id}/stop` | 停止执行中的工作流 | 仅streaming模式支持 |

#### API调用示例

```bash
# 执行工作流（blocking模式）
curl --location --request POST 'https://api.dify.ai/v1/workflows/run' \
--header 'Authorization: Bearer YOUR-API-KEY' \
--header 'Content-Type: application/json' \
--data-raw '{
    "inputs": {
        "user_query": "查询信贷审批流程"
    },
    "response_mode": "blocking",
    "user": "user-123"
}'
```

```python
# Python调用示例
import requests
import json

url = 'https://api.dify.ai/v1/workflows/run'
headers = {
    'Authorization': 'Bearer YOUR-API-KEY',
    'Content-Type': 'application/json'
}
data = {
    "inputs": {"user_query": "查询信贷审批流程"},
    "response_mode": "blocking",
    "user": "user-123"
}
response = requests.post(url, headers=headers, data=json.dumps(data))
print(response.json())
```

#### 响应模式

| 模式 | 说明 | 适用场景 |
|-----|------|---------|
| **blocking** | 同步阻塞，等待完整响应 | 简单查询、短时任务 |
| **streaming** | SSE流式输出，实时返回 | 长文本生成、实时对话 |

#### 长时任务处理

对于耗时较长的工作流，推荐使用异步模式：

```bash
# 启动工作流，获取workflow_run_id
curl -X POST "https://api.dify.ai/v1/workflows/run" \
-H "Authorization: Bearer YOUR-API-KEY" \
-H "Content-Type: application/json" \
-d '{"inputs": {...}, "response_mode": "blocking"}'

# 轮询状态
curl -X GET "https://api.dify.ai/v1/workflows/run/{workflow_run_id}" \
-H "Authorization: Bearer YOUR-API-KEY"
```

工作流状态包括：`running`, `succeeded`, `failed`, `stopped`, `partial-succeeded`

### 4.1 Docker Compose部署（L1验证期）

#### 快速部署步骤

```bash
# 1. 克隆仓库
git clone https://github.com/langgenius/dify.git
cd dify/docker

# 2. 复制环境配置
cp .env.example .env

# 3. 启动服务（需Docker Compose 2.24.0+）
docker compose up -d

# 4. 检查状态
docker compose ps
```

#### 核心容器组件

| 容器 | 镜像 | 端口 | 功能 |
|-----|------|------|------|
| api | langgenius/dify-api | 5001 | 后端API服务 |
| web | langgenius/dify-web | 3000 | 前端界面 |
| db_postgres | postgres:15-alpine | 5432 | PostgreSQL数据库 |
| redis | redis:6-alpine | 6379 | Redis缓存 |
| weaviate | semitechnologies/weaviate | 8080 | 向量数据库 |

### 5.4 Java集成示例

```java
// Spring Boot集成Dify API
@Service
public class DifyWorkflowService {

    private final String DIFY_API_URL = "http://dify-api:5001/v1/workflows/run";
    private final String API_KEY = "your-api-key";

    public WorkflowResult executeWorkflow(String workflowId, Map<String, Object> inputs) {
        RestTemplate restTemplate = new RestTemplate();
        
        HttpHeaders headers = new HttpHeaders();
        headers.set("Authorization", "Bearer " + API_KEY);
        headers.setContentType(MediaType.APPLICATION_JSON);
        
        Map<String, Object> body = new HashMap<>();
        body.put("inputs", inputs);
        body.put("response_mode", "blocking");
        body.put("user", "system");
        
        HttpEntity<Map<String, Object>> request = new HttpEntity<>(body, headers);
        ResponseEntity<WorkflowResult> response = restTemplate.postForEntity(
            DIFY_API_URL, request, WorkflowResult.class
        );
        
        return response.getBody();
    }
}
```

### 7.2 工作流模板示例

#### 制度问答模板

```yaml
# 工作流配置（YAML格式导出）
name: 制度问答工作流
nodes:
  - type: start
    inputs:
      - name: user_query
        type: string
        required: true
  
  - type: knowledge_retrieval
    knowledge_base: 制度库
    query: "{{user_query}}"
    top_k: 5
    score_threshold: 0.7
  
  - type: llm
    model: deepseek-chat
    prompt: |
      基于以下知识库内容回答用户问题：
      {{knowledge_retrieval.output}}
      
      问题：{{user_query}}
      
      要求：
      1. 仅使用知识库内容回答
      2. 标注引用来源
      3. 如无相关内容，回复"未找到相关制度"
  
  - type: end
    outputs:
      - name: answer
        value: "{{llm.output}}"
```

#### 信贷尽调报告模板

```yaml
name: 信贷尽调报告生成
nodes:
  - type: start
    inputs:
      - name: enterprise_name
        type: string
        required: true
  
  - type: http_request
    method: POST
    url: "{{data_platform_api}}/query"
    body:
      enterprise: "{{enterprise_name}}"
    outputs:
      - financial_data
  
  - type: http_request
    method: GET
    url: "{{business_info_api}}/{{enterprise_name}}"
    outputs:
      - company_info
  
  - type: llm
    model: qwen-plus
    prompt: |
      根据以下信息生成信贷尽调报告初稿：
      
      企业名称：{{enterprise_name}}
      财务数据：{{financial_data}}
      工商信息：{{company_info}}
      
      按标准模板格式输出，标注待复核点。
  
  - type: end
    outputs:
      - name: report
        value: "{{llm.output}}"
```

## 九、实施建议与风险预案

### 9.1 分阶段实施路线

| 时间节点 | 里程碑 | 交付物 |
|---------|-------|--------|
| Month 1-2 | L1验证期 | Dify单节点部署、制度问答Demo、网关对接 |
| Month 2-4 | L2应用期 | 工作流模板库建设、知识库导入、业务培训 |
| Month 4-6 | L3生产期 | K8s高可用部署、API开放、多场景上线 |

### 9.2 人力与技能要求

| 角色 | 技能要求 | 建议人数 |
|-----|---------|---------|
| **平台运维** | Docker/K8s、Linux运维 | 1-2人 |
| **工作流设计** | Dify操作、Prompt工程 | 2-3人（业务部门兼职） |
| **知识库管理** | 文档处理、RAG调优 | 1人 |

### 9.3 风险提示与应对

| 风险类型 | 表现 | 应对措施 |
|---------|------|---------|
| **平台锁定** | 无法迁移到其他平台 | 选择开源平台，工作流导出为JSON标准格式 |
| **版本兼容** | 升级后功能异常 | 测试环境验证后再升级生产 |
| **性能瓶颈** | 大并发响应慢 | K8s水平扩展、优化向量检索 |
| **知识库质量** | 检索结果不准确 | 调整切片策略、启用Rerank |

### 9.4 备选方案

如Dify不满足需求，可考虑：

| 备选平台 | 适用场景 |
|---------|---------|
| **n8n** | 业务流程自动化、与现有系统集成 |
| **LangFlow** | 技术团队深度定制、复杂RAG管道 |
| **Harness SDK** | 代码级开发、复杂逻辑、高性能要求 |

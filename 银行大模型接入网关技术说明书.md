# 银行大模型接入网关技术说明书

## 一、项目概述

### 1.1 项目背景

本项目（`/data/guardrails/`）是实现《银行大模型本地化落地与AI转型全景规划书》中"三大硬核控险机制"的技术基础设施。规划书提出的核心风险控制需求：

1. **隐私数据动态脱敏机制** - 在业务系统将请求发给模型前，自动识别并遮掩客户身份证、手机号、账户等敏感隐私信息（PII）
2. **IAM权限强制继承机制** - 大模型检索数据中台时必须强行继承员工原有操作权限，防止越权访问
3. **输入/输出双向实时过滤护栏** - 实时拦截恶意提示词攻击，过滤模型幻觉输出

本项目采用 **Two-Layer Guardrail Architecture** 实现上述机制，为全行大模型应用提供统一的安全入口。

### 1.2 核心价值

| 价值维度 | 具体收益 |
|---------|---------|
| **合规保障** | 从源头免疫隐私泄露红线，满足《个人信息保护法》要求 |
| **风险防控** | 技术硬护栏代偿行政管理组织的缺失，提供刚性的安全承诺 |
| **效率提升** | 统一网关接口，业务部门无需各自开发安全模块 |
| **算力集约** | 集中式处理，避免各应用重复部署安全检测模型 |

### 1.3 技术选型

| 技术组件 | 选型方案 | 选型理由 |
|---------|---------|---------|
| **PII检测框架** | Microsoft Presidio | 金融行业标准，支持自定义识别器扩展 |
| **NLP引擎** | spaCy (en_core_web_sm) | 轻量级，内网离线部署友好 |
| **HTTP框架** | FastAPI + Uvicorn | 高性能异步，原生支持 SSE 流式响应 |
| **HTTP客户端** | httpx | 异步支持，HTTP2 提升连接效率 |
| **配置管理** | Pydantic + YAML | 类型安全，支持环境变量注入 |
| **Layer 2 Judge** | Qwen3Guard-8B-Stream | 中文语义理解能力强，支持快速检测模式 |

### 2.2 Two-Layer Guardrail 设计

```

      │                │                     │                    │
      │                ▼                     ▼                    │
      │         ┌────────────┐        ┌────────────┐             │
      │         │ PII脱敏    │        │ 风险检测   │             │
      │         │ 如：<手机号>│        │ 高风险阻断 │             │
      │         └────────────┘        └────────────┘             │

      │         │ SSE过滤    │        │ 流式拦截   │◀────────────┘
      │         │ (PII脱敏)  │        │ Token级熔断│
      │         └────────────┘        └────────────┘
      │                │                     │
      └────────────────┘◀────────────────────┘
```

**Layer 1: PII规则检测**（默认启用）
- 精准、快速（<1ms）
- 基于正则表达式 + Microsoft Presidio
- 支持简体中文、繁体中文、英文及混合文本
- 自动语言检测，动态选择占位符

**Layer 2: LLM Judge**（可选启用，`judge.enabled: true`）
- 语义理解、意图判断
- 基于 Qwen3Guard-8B-Stream
- 支持流式 Token 级实时拦截
- 风险等级判定：safe → low → medium → high → critical

### 2.3 解耦式护栏架构原则（避免耦合陷阱）

**核心设计思想**：边缘治理、算力解耦。

本网关采用"边缘治理、算力解耦"的设计思想，PII隐私脱敏与输出防幻觉审核完全独立于推理引擎生命周期之外。坚决摒弃将安全逻辑嵌入模型推理节点内部的缺陷路径。

**架构原则说明**：

| 原则 | 说明 | 技术实现 |
|-----|------|---------|
| **护栏外置** | 安全检测逻辑在网关层独立运行 | 网关通过 HTTP/gRPC 与推理引擎通信 |
| **算力隔离** | GPU 推理集群 100% 专注算力满载 | 安全队列拥塞不影响底层推理吞吐 |
| **异步处理** | 安全检测与推理请求异步并行 | Layer 1 同步 + Layer 2 异步可选 |
| **故障边界清晰** | 网关故障不影响推理引擎 | 网关熔断时推理引擎继续运行 |

**缺陷路径警示**：

曾有技术尝试将"防幻觉"或"内容审核"直接写入 vLLM 等推理引擎的自定义 Handler 内部。这种路径已被证明存在严重缺陷：

| 缺陷路径 | 问题说明 | 影响 |
|---------|---------|------|
| 安全逻辑嵌入推理引擎 | 成倍拖慢底层算力的 Token 吞吐率（TTFT） | 性能下降 30-50% |
| 框架耦合 | 推理引擎升级频繁，安全模块极难维护 | 维护成本激增 |
| 资源竞争 | 安全检测与推理共享 GPU 显存 | 显存碎片化，吞吐不稳定 |

**正确架构对比**：

```
❌ 错误路径（耦合架构）：
┌─────────────────────────────────────────────────┐
│           vLLM 推理引擎容器                      │
│  ┌─────────────┐  ┌─────────────┐              │
│  │ 模型推理    │  │ 安全Handler │ ← 资源竞争！  │
│  │ (GPU算力)   │  │ (嵌入内部)  │              │
│  └─────────────┘  └─────────────┘              │
└─────────────────────────────────────────────────┘

✓ 正确路径（解耦架构）：
┌────────────────────┐     HTTP/gRPC     ┌────────────────────┐
│   安全网关容器     │ ────────────────▶ │  推理引擎容器      │
│ ┌────────────────┐ │     异步通信      │ ┌────────────────┐ │
│ │ PII检测        │ │                  │ │ 纯粹推理       │ │
│ │ Judge裁判      │ │                  │ │ (GPU满载)      │ │
│ │ 流式拦截       │ │                  │ └────────────────┘ │
│ └────────────────┘ │                  │     CPU/内存独立   │
│     CPU密集型      │                  └────────────────────┘
└────────────────────┘                        GPU 硬绑定
```

**战略意义**：通过将护栏逻辑剥离至异步网关，确保底层 GPU 推理集群能够 100% 专注于算力满载运行，避免因安全队列拥塞反向拖垮底层大模型的高并发吞吐。这是保障百万级 Token 满载运行的最佳实践。

### 3.2 识别算法详解

**识别器结构**（`chinese_pii_recognizers.py`）：

```python
class ChinaMobilePhoneRecognizer(PatternRecognizer):
    """中国手机号识别器"""
    
    PATTERNS = [
        Pattern(
            name="china_mobile",
            regex=r"(?<!\d)(1[3-9]\d{9})(?!\d)",  # 前后无数字，防止误判
            score=0.95  # 高置信度
        )
    ]
    
    # 上下文关键词（简体+繁体+英文）
    CONTEXT = [
        "手机", "电话", "联系电话",        # 简体
        "手機", "電話", "聯絡電話",        # 繁体
        "mobile", "phone"                  # 英文
    ]
    
    def __init__(self):
        super().__init__(
            supported_entity="CN_PHONE_NUMBER",
            patterns=self.PATTERNS,
            context=self.CONTEXT,
            supported_language="en"  # 使用 "en" 以兼容 Presidio
        )
```

**识别流程**：

```
输入文本 → AnalyzerEngine.analyze() → PatternRecognizer匹配
                    ↓
            上下文增强（CONTEXT关键词）
                    ↓
            置信度评分（score × context_boost）
                    ↓
            阈值过滤（min_score >= 0.5）
                    ↓
            返回 PIIEntity 列表
```

**上下文增强机制**：
- 当文本中出现 CONTEXT 关键词时，置信度提升 0.1-0.2
- 例如：`"手机号：13812345678"` → 检测到"手机号"关键词 → score 从 0.95 提升至 1.0
- 防止误判：`"订单号：13812345678"` → 无上下文关键词 → score 保持 0.95（但可能因订单号上下文被抑制）

### 3.4 语言检测机制

**自动检测逻辑**（`UniversalPIIGuardrail._detect_script()`）：

```python
def _detect_script(self, text: str) -> str:
    # 1. 统计繁简特有字符数量
    traditional_chars = set('們個時說國過...')
    simplified_chars = set('们个时说国过...')
    
    tc_count = sum(1 for c in text if c in traditional_chars)
    sc_count = sum(1 for c in text if c in simplified_chars)
    
    # 2. 判断规则
    if tc_count > sc_count:
        return "traditional"
    elif sc_count > tc_count:
        return "simplified"
    else:
        # 3. 检查特有关键词
        if any(kw in text for kw in ['手機', '電話', '聯絡']):
            return "traditional"
        return "simplified"
```

**语言与占位符映射**：

| 检测结果 | 使用的占位符表 | 示例 |
|---------|--------------|------|
| `simplified` | `DEFAULT_PLACEHOLDERS` | `<手机号>` |
| `traditional` | `TRADITIONAL_PLACEHOLDERS` | `<手機號>` |
| `english` | `ENGLISH_PLACEHOLDERS` | `<PHONE>` |

### 4.1 Judge模块架构

**核心组件**（`gateway/judge.py`）：

### 4.2 流式拦截器

**Token级实时拦截**（`gateway/stream_interceptor.py`）：

```python
class StreamInterceptor:
    """Token级流式拦截器"""
    
    async def intercept(self, stream: AsyncIterator[str]) -> AsyncIterator[str]:
        buffer = ""
        token_count = 0
        
        async for line in stream:
            text = self._extract_text_from_sse(line)
            if text:
                buffer += text
                token_count += len(text.split())
            
            # 每 N 个 token 检测一次
            if token_count % config.check_interval == 0:
                score = await judge.quick_check(buffer)
                
                if score < config.safety_threshold:
                    # 检测到风险，立即中断
                    yield self._create_interrupt_message(score)
                    return  # 中断流
            
            yield line  # 继续转发
```

**配置参数**：

| 参数 | 默认值 | 说明 |
|-----|-------|------|
| `check_interval` | 5 | 每 N 个 token 检测一次 |
| `safety_threshold` | 0.3 | 安全分数阈值，低于此值触发熔断 |
| `min_tokens_before_check` | 10 | 开始检测前的最小 token 数 |

### 4.3 风险等级判定

| 风险等级 | 说明 | 处理策略 |
|---------|------|---------|
| `safe` | 内容安全 | 正常放行 |
| `low` | 低风险 | 记录日志，放行 |
| `medium` | 中等风险 | 标记审计，放行 |
| `high` | 高风险 | 阻断请求 |
| `critical` | 极高风险 | 立即阻断 + 报警 |

**风险类别**：
- `prompt_injection` - 提示词注入攻击
- `harmful_content` - 有害内容
- `pii_leakage` - PII 泄露风险
- `bias_discrimination` - 偏见歧视
- `illegal_content` - 违法内容

**幻觉检测实现**：

| 检测策略 | 技术方案 | 阈值/规则 | 处理方式 |
|---------|---------|----------|---------|
| **引用溯源验证** | 检查文档ID存在性 | 引用的文档必须存在于知识库 | 不存在则标记幻觉 |
| **置信度评分** | Judge模型评估 | score < 0.7 | 拒绝输出，转人工审核 |
| **数字核验** | 与核心系统数据比对 | 利率/金额误差 ≤ 0.01% | 超出误差范围则拦截 |

```python
# gateway/hallucination_detector.py
class HallucinationDetector:
    """幻觉检测器"""
    
    # 置信度阈值
    CONFIDENCE_THRESHOLD = 0.7
    
    async def detect(self, content: str, context: dict) -> HallucinationResult:
        results = []
        
        # 1. 引用溯源验证
        citations = self._extract_citations(content)
        for citation in citations:
            if not await self._verify_citation(citation, context["knowledge_base"]):
                results.append(f"引用验证失败: {citation}")
        
        # 2. 置信度评分
        judge_result = await self.judge.judge(content)
        if judge_result.confidence < self.CONFIDENCE_THRESHOLD:
            results.append(f"置信度过低: {judge_result.confidence:.2f}")
        
        # 3. 数字核验
        numbers = self._extract_financial_numbers(content)
        for num in numbers:
            if not await self._verify_number(num, context["data_platform"]):
                results.append(f"数字核验失败: {num}")
        
        return HallucinationResult(
            has_hallucination=len(results) > 0,
            issues=results,
            confidence=judge_result.confidence
        )
    
    def _extract_citations(self, content: str) -> List[str]:
        """提取引用标识"""
        # 匹配 [文档ID:xxx] 或 【来源:xxx】 格式
        return re.findall(r'[文档ID|来源][:：]\s*([^\]】]+)', content)
    
    def _extract_financial_numbers(self, content: str) -> List[dict]:
        """提取金融数字（利率、金额）"""
        numbers = []
        # 匹配利率: 年化收益率3.5%、利率4.2%
        rates = re.findall(r'(?:年化收益|收益|利)率\s*[是为]?\s*(\d+\.?\d*)%', content)
        for rate in rates:
            numbers.append({"type": "rate", "value": float(rate)})
        
        # 匹配金额: 100万元、10000元
        amounts = re.findall(r'(\d+\.?\d*)\s*([万亿]?元)', content)
        for amount, unit in amounts:
            numbers.append({"type": "amount", "value": float(amount), "unit": unit})
        
        return numbers
```

```python
# gateway/compliance_rules.py

# 违规营销用语黑名单
MARKETING_BLACKLIST = [
    {"term": "保本保息", "action": "block", "reason": "违反监管规定，禁止承诺保本保息"},
    {"term": "稳赚不赔", "action": "block", "reason": "违规营销用语，禁止承诺收益"},
    {"term": "保证收益", "action": "alert", "reason": "需人工审核，合规风险"},
    {"term": "无风险收益", "action": "block", "reason": "误导性表述，所有投资均有风险"},
    {"term": "内部消息", "action": "block", "reason": "涉及内幕信息，触发合规审计"},
    {"term": "独家内幕", "action": "block", "reason": "涉及内幕信息，触发合规审计"},
]

# 敏感信息关键词
SENSITIVE_KEYWORDS = [
    {"term": "客户隐私", "action": "block", "reason": "可能涉及客户隐私泄露"},
    {"term": "内部机密", "action": "block", "reason": "涉及内部机密信息"},
]

class ComplianceChecker:
    """合规检测器"""
    
    def check(self, content: str) -> ComplianceResult:
        violations = []
        
        # 黑名单检测
        for rule in MARKETING_BLACKLIST + SENSITIVE_KEYWORDS:
            if rule["term"] in content:
                violations.append({
                    "term": rule["term"],
                    "action": rule["action"],
                    "reason": rule["reason"]
                })
        
        return ComplianceResult(
            is_compliant=len(violations) == 0,
            violations=violations
        )
    
    def get_action(self, violations: List[dict]) -> str:
        """根据违规情况确定处理动作"""
        for v in violations:
            if v["action"] == "block":
                return "block"
        if any(v["action"] == "alert" for v in violations):
            return "alert"
        return "pass"
```

```python
    async def guard_output(self, content: str, context: dict) -> GuardResult:

        compliance = self.compliance_checker.check(content)
        action = self.compliance_checker.get_action(compliance.violations)
        
        if action == "block":
            return GuardResult(
                action="block",
                reason="违反合规规定",
                violations=compliance.violations,
                fallback="抱歉，该内容涉及不合规表述，已拦截。"
            )
        elif action == "alert":
            return GuardResult(
                action="alert",
                reason="需人工审核",
                violations=compliance.violations,
                content=content  # 放行但标记
            )
        
        # 3. PII二次检测（防止模型泄露PII）
        pii_entities = self.guardrail.detect(content)
        if pii_entities:
            safe_content = self.guardrail.redact(content)

                action="redact",
                reason="检测到PII，已脱敏",
                content=safe_content
            )
        
        return GuardResult(action="pass", content=content)
```

### 7.3 幻觉检测增强

**规划书原文**：
> 一旦监测到模型生成的文本存在"胡言乱语"或捏造我行不存在的理财条款（即模型幻觉），在网关层直接拦截并报错。

**核心检测策略**：

| 检测策略 | 技术方案 | 阈值/规则 | 处理方式 |
|---------|---------|----------|---------|
| **引用溯源验证** | 检查文档ID存在性 | 引用的文档必须存在于知识库 | 不存在则标记幻觉 |
| **置信度评分** | Judge模型评估 | score < 0.7 | 拒绝输出，转人工审核 |
| **数字核验** | 与核心系统数据比对 | 利率/金额误差 ≤ 0.01% | 超出误差范围则拦截 |

#### 7.3.1 引用溯源验证机制

**检测逻辑**：

```python
# gateway/hallucination_detector.py
class HallucinationDetector:
    """幻觉检测器"""
    
    async def detect(self, content: str, context: dict) -> HallucinationResult:
        results = []
        
        # 1. 引用溯源验证
        citations = self._extract_citations(content)
        for citation in citations:
            if not await self._verify_citation(citation, context["knowledge_base"]):
                results.append(f"引用验证失败: {citation}")
        
        # 2. 置信度评分
        judge_result = await self.judge.judge(content)
        if judge_result.confidence < self.CONFIDENCE_THRESHOLD:
            results.append(f"置信度过低: {judge_result.confidence:.2f}")
        
        # 3. 数字核验
        numbers = self._extract_financial_numbers(content)
        for num in numbers:
            if not await self._verify_number(num, context["data_platform"]):
                results.append(f"数字核验失败: {num}")
        
        return HallucinationResult(
            has_hallucination=len(results) > 0,
            issues=results,
            confidence=judge_result.confidence
        )
    
    def _extract_citations(self, content: str) -> List[str]:
        """提取引用标识 - 支持多种引用格式"""
        citations = []
        
        # 格式1: [文档ID:xxx] 或 【来源:xxx】
        pattern1 = r'[文档ID|来源|出处][:：]\s*([^\]】]+)'
        citations.extend(re.findall(pattern1, content))
        
        # 格式2: 根据《xxx》规定
        pattern2 = r'根据[《「【]([^》」】]+)[》」】]'
        citations.extend(re.findall(pattern2, content))
        
        # 格式3: 参见第xxx条/第xxx页
        pattern3 = r'参见[第]?(\d+)[条|页]'
        citations.extend(re.findall(pattern3, content))
        
        return citations
    
    def _extract_financial_numbers(self, content: str) -> List[dict]:
        """提取金融数字（利率、金额）"""
        numbers = []
        
        # 匹配利率: 年化收益率3.5%、利率4.2%
        rates = re.findall(r'(?:年化收益|收益|利)率\s*[是为]?\s*(\d+\.?\d*)%', content)
        for rate in rates:
            numbers.append({"type": "rate", "value": float(rate)})
        
        # 匹配金额: 100万元、10000元
        amounts = re.findall(r'(\d+\.?\d*)\s*([万亿]?元)', content)
        for amount, unit in amounts:
            numbers.append({"type": "amount", "value": float(amount), "unit": unit})
        
        return numbers
```

#### 7.3.2 置信度评分机制

**置信度综合评估公式**：

```
置信度 = 0.4 × retrieval_confidence + 0.3 × coverage_score + 0.3 × consistency_score
```

| 评分维度 | 说明 | 计算方式 |
|---------|------|---------|
| `retrieval_confidence` | 检索置信度 | 检索结果的平均相似度得分 |
| `coverage_score` | 覆盖度评分 | 回答内容与检索内容的匹配度 |
| `consistency_score` | 一致性评分 | 多文档信息交叉验证一致性 |

**置信度阈值配置（按场景）**：

| 业务场景 | 置信度阈值 | 低置信度处理 | 最少引用来源 |
|---------|-----------|-------------|------------|
| **高风险决策**（信贷审批） | ≥ 0.85 | 转人工审核 | ≥ 3个文档 |
| **业务咨询**（产品解答） | ≥ 0.70 | 返回保守回答 + 提示核实 | ≥ 2个文档 |
| **制度查询**（流程说明） | ≥ 0.50 | 标注置信度 + 正常返回 | ≥ 1个文档 |

#### 7.3.3 数字核验机制

**核验流程**：

```
模型输出数字 ──→ 提取数值+类型 ──→ 调用数据中台API ──→ 比对误差
                      │
                      ▼
              ┌───────────────────────────────────┐
              │ 利率核验: 与当前利率表比对         │
              │ 金额核验: 与产品参数表比对         │
              │ 误差阈值: ≤ 0.01%                 │
              └───────────────────────────────────┘
                      │
                      ▼
              误差超限 → 拦截 + 标记幻觉
              误差合规 → 放行
```

**实现代码**：

```python
# gateway/number_verifier.py
class NumberVerifier:
    """金融数字核验器"""
    
    ERROR_THRESHOLD = 0.0001  # 误差阈值 0.01%
    
    async def verify_rate(self, rate_value: float, product_id: str) -> VerifyResult:
        """核验利率"""
        # 从数据中台获取当前利率
        current_rate = await self.data_platform.get_rate(product_id)
        
        if current_rate is None:
            return VerifyResult(
                verified=False,
                reason=f"无法获取产品 {product_id} 的利率信息"
            )
        
        error = abs(rate_value - current_rate) / current_rate
        
        if error > self.ERROR_THRESHOLD:
            return VerifyResult(
                verified=False,
                reason=f"利率误差过大: 声称{rate_value}%, 实际{current_rate}%, 误差{error:.4%}"
            )
        
        return VerifyResult(verified=True)
```

**特征监控机制**：

```python
# gateway/feature_monitor.py
class FeatureScaleMonitor:
    """特征量级监控器 - 保障金融定价模型稳定性"""
    
    # 特征量级参考范围
    FEATURE_SCALE_RANGES = {
        "text_embedding": {"min": -1.0, "max": 1.0, "typical_std": 0.3},
        "interest_rate": {"min": 0.0, "max": 0.25, "typical_std": 0.05},  # 利率：0-25%
        "exchange_rate": {"min": 0.5, "max": 10.0, "typical_std": 1.0},   # 汇率
        "credit_score": {"min": 300, "max": 850, "typical_std": 100},      # 信用分
        "macro_indicator": {"min": -100, "max": 100, "typical_std": 20},   # 宏观指标
    }
    
    def check_feature_scale(
        self,
        feature_name: str,
        feature_value: float
    ) -> ScaleCheckResult:
        """检查特征量级是否在合理范围"""
        
        ref = self.FEATURE_SCALE_RANGES.get(feature_name)
        if not ref:
            return ScaleCheckResult(valid=True, warning=None)
        
        # 检查是否超出典型范围
        if feature_value < ref["min"] or feature_value > ref["max"]:
            return ScaleCheckResult(
                valid=False,
                warning=f"特征 {feature_name} 值 {feature_value} 超出合理范围 [{ref['min']}, {ref['max']}]"
            )
        
        # 检查量级差异（与文本特征相比）
        if feature_name in ["interest_rate", "macro_indicator"]:
            text_scale = self.FEATURE_SCALE_RANGES["text_embedding"]["typical_std"]
            feature_scale = ref["typical_std"]
            scale_ratio = feature_scale / text_scale
            
            if scale_ratio > 10:
                return ScaleCheckResult(
                    valid=True,
                    warning=f"特征 {feature_name} 量级（{feature_scale}）与文本特征（{text_scale}）差异较大，"
                           f"建议在模型训练时进行归一化处理"
                )
        
        return ScaleCheckResult(valid=True, warning=None)
```

```
                 ▼              ▼              ▼
           ┌──────────┐   ┌──────────┐   ┌──────────────┐
           │ 自动生成 │   │ 权限等级 │   │ chunk_id关联 │
           │ doc_id   │   │ 标注     │   │ doc_id       │
           └──────────┘   └──────────┘   └──────────────┘
```

**实现代码**：

```python
# rag/document_processor.py
class DocumentProcessor:
    """文档入库处理器"""
    
    async def process_document(self, raw_doc: dict) -> ProcessedDocument:
        """处理原始文档，生成标准化入库格式"""
        
        # 1. 生成唯一文档ID
        doc_id = self._generate_doc_id(raw_doc)
        
        # 2. 解析文档内容
        content = self._parse_content(raw_doc)
        
        # 3. 标注权限等级
        classification = self._determine_classification(raw_doc)
        
        # 4. 分块处理（保留doc_id关联）
        chunks = self._split_into_chunks(content, doc_id)
        
        return ProcessedDocument(
            doc_id=doc_id,
            doc_type=raw_doc.get("type", "unknown"),
            classification=classification,
            version=raw_doc.get("version", "1.0"),
            effective_date=raw_doc.get("effective_date"),
            expiry_date=raw_doc.get("expiry_date"),
            title=raw_doc.get("title"),
            content=content,
            chunks=chunks  # 每个 chunk 包含 chunk_id 和 doc_id
        )
    
    def _generate_doc_id(self, doc: dict) -> str:
        """生成唯一文档ID"""
        # 格式: KB-{type}-{year}-{sequence}
        doc_type = doc.get("type", "GEN")
        year = datetime.now().year
        sequence = self._get_next_sequence(doc_type)
        return f"KB-{doc_type}-{year}-{sequence:04d}"
    
    def _split_into_chunks(self, content: str, doc_id: str) -> List[Chunk]:
        """分块处理，每个chunk关联doc_id"""
        paragraphs = self._split_paragraphs(content)
        chunks = []
        
        for i, para in enumerate(paragraphs):
            chunks.append(Chunk(
                chunk_id=f"{doc_id}-C{i+1:03d}",
                doc_id=doc_id,
                content=para,
                position=i + 1
            ))
        
        return chunks
```

```python
            f"[文档ID:{self.doc_id}] [置信度:{confidence}] [相关性:{self.score:.2f}]\n"

            f"内容: {self.content}\n"

        if score >= 0.85:
            return "高置信"
        elif score >= 0.70:
            return "中置信"
        elif score >= 0.50:
            return "低置信"
        else:
            return "不可信"
```

### 8.2 置信度评分的前置改造

**改造目标**：确保置信度评分能够综合检索、覆盖、一致性三个维度。

#### 8.2.1 检索置信度传递机制

**RAG检索结果需携带置信度信息**：

```python
# rag/confidence_evaluator.py
class RAGConfidenceEvaluator:
    """RAG回答置信度评估器"""
    
    async def evaluate_confidence(
        self,
        query: str,
        retrieval_results: List[RAGSearchResult],
        model_response: str
    ) -> ConfidenceReport:
        """
        综合评估回答置信度
        
        置信度 = 0.4 × retrieval_confidence + 0.3 × coverage_score + 0.3 × consistency_score
        """
        
        # 1. 检索置信度（平均检索得分）
        retrieval_confidence = self._calc_retrieval_confidence(retrieval_results)
        
        # 2. 覆盖度评分（回答内容与检索内容的匹配度）
        coverage_score = await self._calc_coverage_score(model_response, retrieval_results)
        
        # 3. 一致性评分（多文档信息是否一致）
        consistency_score = await self._calc_consistency_score(retrieval_results)
        
        # 综合置信度
        overall_confidence = (
            0.4 * retrieval_confidence +
            0.3 * coverage_score +
            0.3 * consistency_score
        )
        
        return ConfidenceReport(
            overall=overall_confidence,
            retrieval=retrieval_confidence,
            coverage=coverage_score,
            consistency=consistency_score,
            num_sources=len(retrieval_results),
            details=self._generate_details(retrieval_results)
        )
    
    def _calc_retrieval_confidence(self, results: List[RAGSearchResult]) -> float:
        """计算检索置信度"""
        if not results:
            return 0.0
        
        # 加权平均：前3个结果权重更高
        weights = [0.4, 0.3, 0.2, 0.1]  # 第1-4个结果的权重
        total = 0.0
        weight_sum = 0.0
        
        for i, r in enumerate(results[:4]):
            w = weights[i] if i < len(weights) else 0.05
            total += r.score * w
            weight_sum += w
        
        return total / weight_sum if weight_sum > 0 else 0.0
    
    async def _calc_coverage_score(
        self,
        response: str,
        results: List[RAGSearchResult]
    ) -> float:
        """计算覆盖度评分"""
        # 提取回答中的关键陈述
        statements = self._extract_key_statements(response)
        
        if not statements:
            return 1.0  # 无具体陈述视为全覆盖
        
        covered = 0
        for stmt in statements:
            # 检查每个陈述是否有检索内容支撑
            for r in results:
                if self._is_supported(stmt, r.content):
                    covered += 1
                    break
        
        return covered / len(statements)
    
    async def _calc_consistency_score(self, results: List[RAGSearchResult]) -> float:
        """计算一致性评分"""
        if len(results) <= 1:
            return 1.0
        
        # 提取各文档的关键信息
        key_infos = []
        for r in results:
            key_infos.append(self._extract_key_numbers(r.content))
        
        # 检查关键信息是否一致
        inconsistencies = self._check_inconsistencies(key_infos)
        
        # 一致性评分 = 1 - 不一致数量/比较次数
        total_checks = len(key_infos) * (len(key_infos) - 1) / 2
        consistency = 1 - len(inconsistencies) / max(total_checks, 1)
        
        return max(0.0, consistency)
```

#### 8.2.2 低置信度降级处理

**降级流程设计**：

```
置信度评估 ──→ 判断阈值 ──→ 处理分支
                    │
        ┌───────────┼───────────┐
        │           │           │
        ▼           ▼           ▼
   ≥ 0.85      0.70-0.85    < 0.70
   高置信      中置信        低置信
   正常返回    标注返回      降级处理
        │           │           │
        ▼           ▼           ▼
   完整回答   回答+提示    保守回答
              "请核实"    或转人工
```

**降级处理实现**：

```python
# gateway/response_handler.py
class ResponseHandler:
    """响应处理器（含降级机制）"""
    
    async def handle_response(
        self,
        model_response: str,
        confidence: ConfidenceReport,
        scenario: str = "business_query"
    ) -> FinalResponse:
        """根据置信度处理响应"""
        
        threshold = self._get_threshold(scenario)
        
        if confidence.overall >= threshold:
            # 高置信度：正常返回
            return FinalResponse(
                content=model_response,
                confidence=confidence.overall,
                confidence_label="高置信",
                citations=self._extract_citations(model_response),
                action="pass"
            )
        
        elif confidence.overall >= 0.70:
            # 中置信度：标注返回
            warning = "\n\n【提示】该回答置信度为中等（{:.2f}），建议核实关键信息后再做决策。".format(confidence.overall)
            return FinalResponse(
                content=model_response + warning,
                confidence=confidence.overall,
                confidence_label="中置信",
                citations=self._extract_citations(model_response),
                action="warn"
            )
        
        else:
            # 低置信度：降级处理
            fallback = self._get_fallback_message(scenario)
            return FinalResponse(
                content=fallback,
                confidence=confidence.overall,
                confidence_label="低置信",
                citations=[],
                action="fallback",
                suggest_human=True
            )
    
    def _get_fallback_message(self, scenario: str) -> str:
        """获取降级消息"""
        fallbacks = {
            "risk_decision": "抱歉，系统无法确认该信息的准确性。根据风险控制要求，此问题需转人工审核。请联系您的专属客户经理或拨打客服热线。",
            "business_query": "抱歉，系统对该问题的回答置信度较低。建议您咨询人工客服或查阅官方渠道信息以获取准确答复。",
            "policy_lookup": "根据现有资料，系统无法完全确认该信息。建议您查阅相关规章制度原文或咨询合规部门。"
        }
        return fallbacks.get(scenario, fallbacks["business_query"])
```

### 9.1 容器化部署与算力隔离规范

**部署原则**：鉴于我行全物理隔离内网对运行确定性的极端要求，底层推理引擎与安全网关必须彻底解耦。

#### 9.1.1 推理引擎部署规范

底层推理引擎（如 vLLM/TensorRT-LLM）将坚决**规避使用 Docker Compose 等微服务编排工具**，一律采取**原生 Docker 独立运行（Direct Docker run）**模式。

**部署规范说明**：

| 规范项 | 要求 | 原因 |
|-------|------|------|
| **禁止编排工具** | 不使用 Docker Compose/Kubernetes | 避免编排层网络自动发现引发的隐形不确定性 |
| **原生 Docker run** | 使用精简的容器化原子脚本 | 确保故障边界清晰化 |
| **GPU 硬绑定** | 显式指定 `--gpus device=X` | 算力资源绝对硬绑定，避免资源争抢 |
| **共享内存配置** | 显式指定 `--shm-size` | 防止大模型推理时共享内存不足 |

### 9.4 配置文件详解

**关键配置项说明**：

```yaml
# filter.enabled - 是否启用 PII 过滤
#   true: 所有请求经过 PII 检测和脱敏
#   false: 直接转发请求（不推荐）
filter:
  enabled: true

# filter.min_score - PII 检测置信度阈值
#   0.5: 默认值，平衡准确率和召回率
#   0.7: 高阈值，减少误报，可能漏检
#   0.3: 低阈值，减少漏检，可能误报
  min_score: 0.5

# filter.filter_response - 是否过滤 LLM 响应
#   false: 只过滤用户输入（推荐）
#   true: 同时过滤 LLM 输出（可能影响输出质量）
  filter_response: false

# judge.enabled - 是否启用 Layer 2 Judge
#   false: 只使用 Layer 1 PII 检测（默认）
#   true: 同时启用语义风险检测（需要部署 Judge 服务）
judge:
  enabled: false

# judge.timeout_action - Judge 超时策略
#   "pass": 超时时放行请求（保守策略）
#   "block": 超时时阻断请求（严格策略）
  timeout_action: "pass"

# stream_intercept.safety_threshold - 流式拦截安全阈值
#   0.3: 默认值，较宽松
#   0.5: 中等严格
#   0.7: 高严格，频繁熔断
  safety_threshold: 0.3
```

### 附录B补充：流式拦截（SSE Stream Interruption）应急治理机制

当模型在流式输出过程中触发"输出窗"护栏拦截时，网关将立即执行熔断。网关不会直接向客户端抛出粗暴的 500 错误，而是采用结构化的安全中断标记，确保前端 UI 交互不崩溃的同时，刚性守住合规底线。

**流式拦截处理流程**：

```
流式输出 ──→ Token级检测 ──→ 触发阈值 ──→ 执行熔断
                │
                ▼
         ┌─────────────────────────────────────────────┐
         │         SSE 安全中断注入机制                │
         │                                             │
         │  1. 注入结构化安全标记（不中断SSE连接）      │
         │     data: {"type": "safety_break", ...}     │
         │                                             │
         │  2. 发送用户友好提示                         │
         │     "[内容安全警告：检测到潜在风险内容，     │
         │      输出已熔断。请咨询人工客服。]"         │
         │                                             │
         │  3. 正常结束SSE流                           │
         │     data: [DONE]                            │
         │                                             │
         │  4. 触发人在回路(HITL)机制                  │
         │     会话标记为高风险 + 推送人工审核队列     │
         └─────────────────────────────────────────────┘
```

**SSE中断消息格式**：

```json
// 中断消息（注入到SSE流中）
data: {
  "choices": [{
    "delta": {
      "content": "\n\n【内容安全警告】检测到潜在风险内容，输出已熔断。如需继续咨询，请联系人工客服或拨打客服热线。"
    },
    "finish_reason": "content_filter",
    "index": 0
  }],
  "object": "chat.completion.chunk",
  "safety_info": {
    "interrupted": true,
    "reason": "risk_content_detected",
    "risk_level": "high",
    "confidence": 0.25
  }
}

// 流结束标记
data: [DONE]
```

**人在回路（HITL）机制**：

| 触发条件 | HITL处理动作 | 审计记录 |
|---------|-------------|---------|
| 流式输出触发熔断 | 会话ID标记为高风险，推送人工审核队列 | 记录完整对话历史、熔断原因、风险等级 |
| 置信度低于阈值（<0.7） | 标记为待审核，可返回保守回答 | 记录低置信度原因、涉及的关键陈述 |
| 引用验证失败 | 标记为幻觉嫌疑，建议人工核实 | 记录失败的引用ID、回答内容片段 |

**应急治理代码实现**：

```python
# gateway/stream_emergency.py
class StreamEmergencyHandler:
    """流式输出应急治理处理器"""
    
    def create_safety_interrupt_message(
        self,
        reason: str,
        risk_level: str,
        partial_content: str
    ) -> str:
        """创建结构化的安全中断SSE消息"""
        
        # 用户友好提示（不暴露技术细节）
        user_message = self._get_user_friendly_message(risk_level)
        
        # 构建SSE数据包
        safety_info = {
            "interrupted": True,
            "reason": reason,
            "risk_level": risk_level,
            "timestamp": datetime.now().isoformat()
        }
        
        interrupt_chunk = {
            "choices": [{
                "delta": {"content": user_message},
                "finish_reason": "content_filter",
                "index": 0
            }],
            "object": "chat.completion.chunk",
            "safety_info": safety_info
        }
        
        return f"data: {json.dumps(interrupt_chunk, ensure_ascii=False)}\n\n"
    
    def _get_user_friendly_message(self, risk_level: str) -> str:
        """获取用户友好的中断提示"""
        messages = {
            "high": "\n\n【内容安全提示】系统检测到该回复可能包含敏感信息，已自动中断。如有疑问，请咨询人工客服。",
            "medium": "\n\n【内容提示】该回复置信度较低，建议您核实关键信息或咨询人工客服。",
            "low": "\n\n【提示】该回复仅供参考，建议查阅官方渠道获取准确信息。"
        }
        return messages.get(risk_level, messages["medium"])
    
    async def trigger_hitl_audit(
        self,
        session_id: str,
        conversation_history: List[dict],
        interrupt_reason: str,
        risk_level: str
    ) -> None:
        """触发人在回路（HITL）审计机制"""
        
        audit_record = {
            "session_id": session_id,
            "timestamp": datetime.now().isoformat(),
            "risk_level": risk_level,
            "interrupt_reason": interrupt_reason,
            "conversation_history": conversation_history,
            "status": "pending_review",
            "assigned_to": None  # 待分配审核人员
        }
        
        # 推送到人工审核队列
        await self.audit_queue.push(audit_record)
        
        # 发送审计告警（可选）
        if risk_level == "high":
            await self.alert_system.send_high_risk_alert(audit_record)
```
